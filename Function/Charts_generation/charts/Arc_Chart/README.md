# Arc Chart

图表分类: 关系图表  
感知排名: ⭐⭐⭐☆☆

## 简介

弧图（Arc Chart）用**弧线连接节点**来展示关系和流向。  
它通过将节点沿水平轴排列，用弧线的粗细和高度表示关系强度，使用户可以观察：

- 不同节点之间的关系强度
- 流向和迁移的方向与规模
- 多个节点间的复杂关系网络

相比桑基图，弧图更适合展示**双向关系**和**节点间的对称性**。


## 数据格式

需要包含以下字段：

- source列（起点）：表示弧线的起始节点
- target列（终点）：表示弧线的目标节点
- value列（数值）：表示关系强度（弧线粗细和高度）

推荐数据结构如下：

| source | target | value |
|--------|--------|-------|
| 北美洲 | 拉丁美洲 | 900 |
| 北美洲 | 亚洲 | 600 |
| 北美洲 | 欧洲 | 500 |
| 拉丁美洲 | 北美洲 | 800 |
| 拉丁美洲 | 欧洲 | 400 |
| 亚洲 | 欧洲 | 350 |


## 适用场景

弧图适用于以下场景：

- 展示国际移民或贸易流向
- 比较城市间的人口迁移
- 分析关系强度和流向
- 展示双向关系网络
- 强调节点间的连接关系


## 典型场景

- 国际移民流向分析
- 城市间人口迁移
- 国家间贸易流向
- 部门间的资源流动
- 用户行为转移分析


## 使用建议

- 节点数量建议不超过 20 个，否则图表会显得拥挤
- 弧线数量建议不超过 200 条，可通过 `top_n` 参数过滤
- 节点顺序会影响视觉效果，建议按逻辑排列（如地理位置或流量大小）
- 弧线粗细与 value 值成正比，建议数据范围在 100-10000 之间
- 可通过 `min_value` 或 `min_percentile` 参数过滤小值关系
- 使用 `label_top_n` 参数控制显示的节点标签数量


## 注意事项

- source 和 target 应为离散类别字段
- value 应为数值型字段，且为正数
- 自环（source == target）会被自动过滤
- 若节点过多或边过多，建议使用过滤参数减少复杂度
- 弧线透明度和粗细可通过 options 参数调整


## 与相似图表的区别

### 弧图 vs 桑基图

- 弧图：节点沿水平轴排列，弧线连接，适合展示双向关系
- 桑基图：节点分层排列，流向从左到右，适合展示单向流动

### 弧图 vs 网络图

- 弧图：节点固定在水平轴上，关系用弧线表示
- 网络图：节点自由分布，关系用连线表示，更适合复杂网络

### 弧图 vs 弦图

- 弧图：节点在轴上，弧线在上方
- 弦图：节点在圆周上，弦线在圆内，更适合展示循环关系
