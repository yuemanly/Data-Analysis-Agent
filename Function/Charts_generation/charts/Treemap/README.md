# 矩形树图 Treemap

**图表分类**: 占比图 PART-TO-WHOLE  
**感知排名**: ⭐⭐⭐⭐☆

## 简介

矩形树图通过矩形的面积编码数值大小，用于展示有层级关系的分类数据。相比饼图，树图能展示更多类别且支持多层级嵌套；相比柱状图，树图更节省空间。适合展示嵌套分类数据（如品类→子品类→SKU）或大型目录结构。

**核心特点**：
- 用矩形面积表示占比，支持多层级嵌套展示
- 适合展示有层级且数量较多的分类数据
- 比柱状图更节省空间，比饼图支持更多类别
- 支持交互式钻取和悬停详情

## 数据格式

需要包含以下字段：
- labels：矩形标签名称，每个矩形的唯一标识
- values ：矩形面积权重，必须为正数

可选列：
- parents：父级标签，用于构建层级关系。父级标签必须在 labels 列中存在

**简单树图（无层级）**：

## 适用场景

✅ **典型场景**
- 嵌套分类展示：品类→子品类→SKU（电商产品结构）
- 组织结构展示：公司→部门→团队
- 文件系统展示：磁盘→文件夹→文件
- 预算分配：总预算→部门→项目
- 市场份额：市场→地区→品牌

✅ **使用建议**
1. 数值必须为正数（>0）
2. 行数建议≤200（过多先聚合）
3. 层级深度建议≤4层（过深用旭日图）
4. 支持多层级嵌套，通过 parents 列定义层级关系
5. 颜色自动按层级或数值大小分配

## 注意事项

- 数值必须>0：负数或零值会被自动过滤，返回警告
- 层级关系：parents 列中的值必须在 labels 列中存在（形成有效的树结构）
- 性能：数据行数过多（>500）会影响渲染性能，建议先聚合
- 标签显示：标签过长会自动截断，悬停显示完整内容

## 与相似图表的区别

### 树图 vs 饼图

| 特性 | 树图 | 饼图 |
|------|------|------|
| 类别数 | 多（10-100+） | 少（2-7） |
| 层级 | 支持多层级 | 仅单层 |
| 空间利用 | 高效 | 浪费 |
| 精度 | 高（面积易比较） | 低（角度难比较） |
| 用途 | 嵌套分类 | 简单占比 |

### 树图 vs 柱状图

| 特性 | 树图 | 柱状图 |
|------|------|--------|
| 类别数 | 多（支持嵌套） | 中等（3-30） |
| 层级 | 支持多层级 | 仅单层 |
| 空间利用 | 高效 | 需要较多空间 |
| 精度 | 中（面积感知） | 高（高度精确） |
| 用途 | 嵌套分类 | 单层对比 |

### 树图 vs 旭日图

| 特性 | 树图 | 旭日图 |
|------|------|--------|
| 形状 | 矩形 | 圆形 | 
| 层级 | 支持多层级 | 支持多层级 |
| 空间利用 | 高效 | 中等 |
| 交互 | 点击钻取 | 点击钻取 |
| 用途 | 通用嵌套展示 | 周期性/圆形数据 |
