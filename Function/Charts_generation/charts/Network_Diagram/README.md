# 网络图 Network Diagram

图表分类: 关系 Relationship | 书章节: Ch8  
感知排名: ⭐⭐⭐⭐☆

## 简介

网络图用节点和边表示实体之间的关系。边的粗细可编码权重（如强度、频率、流量），适合展示社交网络、知识图谱、供应链等多对多的复杂关系。

相比弦图，网络图更灵活；相比桑基图，网络图支持双向关系。

## 数据格式

**必需列**

| 列名 | 类型 | 说明 |
|------|------|------|
| source | 文本/类别 | 关系起点（节点名称） |
| target | 文本/类别 | 关系终点（节点名称） |

**可选列**

| 列名 | 类型 | 说明 |
|------|------|------|
| weight | 数值(>0) | 关系强度（编码边的粗细） |

**示例数据**

| source | target | weight |
|--------|--------|--------|
| Alice | Bob | 5 |
| Alice | Charlie | 3 |
| Bob | Charlie | 8 |
| Bob | David | 2 |

## 适用场景

✅ **典型场景**
- 社交网络：用户之间的关注/好友关系
- 知识图谱：概念之间的关联关系
- 供应链网络：企业之间的合作关系
- 通讯网络：电话/邮件往来频率
- 引用网络：论文之间的引用关系
- 协作网络：团队成员之间的协作强度

✅ **使用建议**
1. 节点数控制在 5-50 个为佳（过多会混乱）
2. 权重值应为正数，会自动归一化
3. 支持自环（同一节点的内部关系）
4. 自动检测并合并重复边
5. 力导向布局自动排列节点，避免重叠

## 注意事项

- 重复边会自动合并（权重求和）
- 零值或负值权重会被过滤
- 孤立节点（无入度或出度）不会显示
- 节点数过多（>100）可能导致布局混乱
- 权重仅影响边的粗细，不影响节点大小

## 与相似图表的区别

### 网络图 vs 弦图

| 特性 | 网络图 | 弦图 |
|------|--------|------|
| 关系方向 | 无向/有向 | 无向（循环） |
| 节点布局 | 力导向（自由） | 圆形（固定） |
| 适用 | 复杂网络 | 循环关系 |
| 可读性 | 高（节点分散） | 中（节点拥挤） |

### 网络图 vs 桑基图

| 特性 | 网络图 | 桑基图 |
|------|--------|--------|
| 流向 | 双向/无向 | 单向 |
| 权重含义 | 关系强度 | 流量大小 |
| 布局 | 力导向 | 分层 |
| 适用 | 关系探索 | 流向展示 |

### 网络图 vs 散点图

| 特性 | 网络图 | 散点图 |
|------|--------|--------|
| 数据类型 | 关系数据 | 数值数据 |
| 维度 | 2D 关系 | 2D/3D 数值 |
| 用途 | 关系可视化 | 相关性分析 |
