# Stacked Bar

图表分类: 比较图表  
感知排名: ⭐⭐⭐⭐☆

## 简介

堆叠柱状图（Stacked Bar Chart）用于展示**多个分组在各类别中的构成比例和总量**。  
它通过将不同分组的柱子堆叠在一起，使用户可以观察：

- 各分组对总量的贡献
- 不同类别间的总量差异
- 各分组在不同类别中的相对大小变化

相比分组柱状图，堆叠柱状图更适合强调**部分与整体的关系**。


## 数据格式

需要包含以下字段：

- x列（类别）：表示横轴分类
- y列（数值）：表示对应的数值大小
- color列（分组）：表示不同的分组类别

或使用**宽格式数据**（自动转换）：

| 行标签 | 标签二 | 标签三 | 标签四 | 标签一 |
|--------|--------|--------|--------|--------|
| 第一周 | 36.39% | 49.72% | 73.50% | 24.53% |
| 第二周 | 31.97% | 44.48% | 69.08% | 23.86% |
| 第三周 | 29.03% | 35.79% | 63.14% | 21.95% |
| 第四周 | 28.11% | 35.32% | 60.85% | 21.24% |
| 第五周 | 25.46% | 30.22% | 56.75% | 13.82% |


## 适用场景

堆叠柱状图适用于以下场景：

- 展示各部分对总体的贡献比例
- 比较不同类别的总量及内部构成
- 分析多个系列在各分类中的变化趋势
- 强调"总量"和"部分构成"的关系
- 展示百分比或比例数据


## 典型场景

- 展示不同产品在各季度的销售额及占比
- 比较不同部门在各月份的预算分配
- 分析不同来源的流量在各时间段的构成
- 展示不同类别的支出在各项目中的分布


## 使用建议

- 当分组数量较少（通常 2~5 个）时效果最佳
- 如果分组过多，图表会显得拥挤，建议改用：
  - 分组柱状图
  - 折线图
  - 分面图
- 建议保持统一的颜色编码，便于识别不同分组
- 对于百分比数据，可以更清晰地展示各部分的占比
- 类别名称较长时可考虑旋转标签或改为横向条形图


## 注意事项

- x应为离散类别字段，不建议直接使用连续数值
- color 字段应为有限个分类值，否则图例会过多
- y应为数值型字段
- 若某些类别下缺少部分分组数据，堆叠会自动调整
- 宽格式数据会自动转换，第一列作为 x 轴，其余列作为分组


## 与相似图表的区别

### 堆叠柱状图 vs 分组柱状图

- 堆叠柱状图：适合查看总量及内部构成
- 分组柱状图：适合比较各分组之间的差异

### 堆叠柱状图 vs 百分比堆叠柱状图

- 堆叠柱状图：显示绝对值，柱子高度代表总量
- 百分比堆叠柱状图：显示相对值，所有柱子高度相同（100%）

### 堆叠柱状图 vs 基础柱状图

- 堆叠柱状图：一个类别中包含多个系列，堆叠显示
- 基础柱状图：每个类别通常只有一个数值系列
