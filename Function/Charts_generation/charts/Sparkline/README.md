# 迷你图

图表分类: 趋势 Trend  
感知排名: ⭐⭐☆☆☆

## 简介

迷你图是一种极简的趋势线条图，专为表格嵌入设计。它为每一行数据生成一个紧凑的趋势迷你图，通过颜色编码快速传达数据的整体趋势方向：

- 为每行数据自动生成一个迷你图
- 用颜色编码趋势方向（绿色上升、红色下降）
- 无坐标轴、无网格、无图例，最大化信息密度
- 支持 hover 查看具体数值

相比完整的折线图，迷你图牺牲了精确性换取极致的紧凑性，适合在表格中并列展示多个实体的趋势。

## 数据格式

支持**宽格式数据**（推荐）：

- 首列：行标签（实体名称、国家、产品等），文本类型
- 其余列：数值序列（时间点或指标），数值类型

典型数据结构：

| Country | 2000 | 2015 |
|---------|------|------|
| Australia | 7.6 | 9.4 |
| Canada | 8.3 | 10.4 |
| Finland | 6.8 | 9.4 |
| Japan | 7.2 | 10.9 |
| Switzerland | 9.3 | 12.1 |
| Turkey | 4.6 | 4.1 |
| United Kingdom | 6.0 | 9.9 |
| United States | 12.5 | 16.8 |

## 适用场景

迷你图适用于以下场景：

- 表格中的趋势列（每行一个迷你图）
- 多个实体的并列趋势对比
- 仪表盘中的多指标展示
- 报表中的快速趋势概览

## 典型场景

- 国家经济指标表中的趋势迷你图
- 产品销售数据表中的月度趋势
- 股票行情表中的日线走势
- 部门业绩表中的季度趋势对比

## 使用建议

- 数据点建议 2-10 个，过少无法体现趋势，过多会显得拥挤
- 与表格结合使用时，每行一个迷你图效果最佳
- 颜色自动根据趋势方向选择（起点→终点：上升绿色、下降红色）
- 支持 hover 查看具体数值，但不建议频繁交互
- 标签列会自动识别为首列，也可通过 mapping 指定

## 注意事项

- 迷你图不适合精确数值读取，仅用于趋势概览
- 不显示坐标轴和具体数值标签，需要通过表格辅助
- 极小的尺寸（60×200px）可能导致细节丢失
- 建议配合表格中的数值列使用，而非单独展示
- 数值列必须是数值类型，文本会被忽略

## 与相似图表的区别

### 迷你图 vs 折线图

- 迷你图：极简设计，无轴无标签，适合嵌入表格，每行一个
- 折线图：完整设计，有轴有标签，适合独立分析，单条线

### 迷你图 vs 面积图

- 迷你图：紧凑型，用于密集展示多个实体的趋势
- 面积图：展示型，用于强调单个或少数几个序列的数值大小和变化

### 迷你图 vs 柱状图

- 迷你图：连续趋势展示，强调变化方向和幅度
- 柱状图：离散数值对比，强调各时间点的绝对值
