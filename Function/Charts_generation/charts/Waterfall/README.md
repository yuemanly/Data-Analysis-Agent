# 瀑布图

图表分类: 累积
感知排名: ⭐⭐⭐⭐☆

## 简介

瀑布图展示数值从起点经过增减变化到终点的过程，通过柱子的堆叠展示累积效果。它通过不同颜色的柱子和连接线，使用户可以直观观察：

- 各阶段对总体的贡献（增加或减少）
- 从初始值到最终值的完整变化过程
- 正向和负向变化的相对大小

相比柱状图，瀑布图更适合展示累积变化过程；相比折线图，瀑布图更适合展示离散阶段的变化分解。


## 数据格式

需要包含以下字段：

- x 列（阶段名称）：各个变化阶段的标签
- y 列（数值）：各阶段的增减值
- type 列（可选）：阶段类型，可选值为 initial/increase（增长）、decrease（下降）、total（总计）

**关键规则**：
- 首行自动识别为 `total`（起始总计，深蓝色）
- 末行自动识别为 `total`（最终总计，深蓝色）
- 中间行根据 type 列或标签关键词判断：
  - 包含"新增"、"增加"、"increase" → `increase`（绿色）
  - 包含"流失"、"减少"、"decrease" → `decrease`（红色）
  - 无关键词时按 y 值正负判断

典型数据结构（无 type 列）：

| 阶段 | 人数   |
|------|------|
| 初始活跃人数 | 378  |
| W16新增 | 397  |
| W16流失 | -378 |
| W17新增 | 394  |
| W17流失 | -397 |
| 最终活跃人数 | 268  |

或带 type 列：

| 阶段 | 金额 | 类型 |
|------|------|------|
| 初始收入 | 1000 | total |
| 成本 | -300 | decrease |
| 税费 | -100 | decrease |
| 最终利润 | 600 | total |


## 适用场景

瀑布图适用于以下场景：

- 利润分解：展示从营收到利润的完整变化过程
- 成本构成分析：展示各项成本对总成本的贡献
- 财务变动分析：展示资产、负债、权益的变化
- 销售漏斗分析：展示从潜在客户到实际销售的转化过程
- 用户活跃度变化：展示用户新增、流失、最终活跃数


## 典型场景

- 年度利润分解：初始收入 → 成本 → 税费 → 最终利润
- 预算执行分析：预算 → 实际支出 → 结余
- 用户转化漏斗：访问 → 注册 → 激活 → 付费 → 留存
- 库存变动：期初库存 → 入库 → 出库 → 期末库存
- 用户活跃度：初始活跃 → 新增 → 流失 → 最终活跃


## 使用建议

- 数据点建议 ≥ 3 个，通常首尾为总计，中间为增减项
- 无需提供 type 列，系统自动推断：首尾为 total，中间根据标签或正负判断
- 如提供 type 列，确保中间行标记为 increase 或 decrease
- 正值显示为绿色（增长），负值显示为红色（下降），总计显示为深蓝
- 标签包含"新增"、"增加"自动识别为增长；包含"流失"、"减少"自动识别为下降


## 注意事项

- 缺失值会被自动删除
- y 列必须可转换为数值类型
- type 列支持中英文混合识别（total/合计/总、increase/增长、decrease/下降）
- 连接线自动连接各柱子，展示累积过程
- 首行和末行始终显示为总计柱（深蓝色），不受 type 列影响


## 与相似图表的区别

### 瀑布图 vs 柱状图

- 瀑布图：展示累积变化过程，柱子之间有连接线，强调增减贡献
- 柱状图：展示独立的数值对比，无连接线，强调绝对值大小

### 瀑布图 vs 堆叠柱状图

- 瀑布图：展示从起点到终点的单一路径，强调变化过程
- 堆叠柱状图：展示多个维度的组成，强调部分与整体关系

### 瀑布图 vs 折线图

- 瀑布图：展示离散阶段的累积变化，强调每个阶段的贡献
- 折线图：展示连续的趋势变化，强调时间序列的演变
