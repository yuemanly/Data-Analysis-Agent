# 动态流向图 Flow Map

图表分类: 地理 Geospatial  
感知排名: ⭐⭐⭐⭐⭐

## 简介

动态流向图用箭头和涟漪效果表示地点间的流向关系。
它通过动画箭头和数据点涟漪，使用户可以：

- 观察地点间的流向强度（箭头方向和流量）
- 识别主要流向枢纽（涟漪密度）
- 追踪动态流向过程（箭头动画）

相比静态流向图，动态流向图更直观、更吸引注意力。

## 数据格式

需要包含以下字段：

- from：起点地区名称（如"北京"、"上海"）
- to：终点地区名称（如"广州"、"成都"）
- value：流量数值（Numeric，如人口流动、商品流量、资金流向）

示例数据结构：

| 起点 | 终点 | 流量 |
|------|------|------|
| 北京 | 广州 | 100 |
| 上海 | 成都 | 150 |
| 广州 | 哈尔滨 | 120 |

## 适用场景

动态流向图适用于以下场景：

- 城市间人口迁移流向
- 商品流通网络（工厂→仓库→门店）
- 资金流向分析（投资、转账）
- 物流配送网络
- 旅游客流分布
- 数据中心间的流量

## 典型场景

- 春运期间全国主要城市人口流向
- 电商物流配送中心间的货物流向
- 跨境电商的商品流向网络
- 金融机构间的资金流向
- 互联网用户的地理迁移流向

## 使用建议

- 数据应为**流向关系**（起点→终点），每行代表一条流向
- 流量数值应为**正数**，表示流向强度
- 地区名需与 pyecharts 内置坐标库匹配（3700+ 中国城市/区县）
- `maptype` 可设为 `china`、省名（如"湖北"）或市名（如"宜昌"）
- 箭头颜色为金色（#FFB81C），数据点为蓝色（#0084D1）
- 涟漪效果自动根据流量累计显示

## 注意事项

- 地名不匹配时该流向跳过并记录 warning
- 数据格式为长格式：每行一条流向
- 同一地点的流量会自动累计（用于涟漪大小）
- 流向线支持双向（A→B 和 B→A 可同时存在）

## 与相似图表的区别

### 动态流向图 vs 静态流向图

| 对比维度 | 动态流向图 | 静态流向图 |
|---------|-----------|----------|
| 视觉编码 | 箭头动画 + 涟漪效果 | 静态线条 |
| 交互性 | ✅ 高（动画吸引注意力） | ❌ 低 |
| 性能 | ⚠️ 中等（动画消耗资源） | ✅ 高 |
| 适用场景 | 展示、演讲、实时监控 | 报告、分析、存档 |
| 可读性 | 直观、生动 | 清晰、精确 |

### 动态流向图 vs 桑基图

| 对比维度 | 动态流向图 | 桑基图 |
|---------|-----------|--------|
| 数据表达 | 地点间的流向 | 多层级的流向转换 |
| 地理信息 | ✅ 包含（基于地图） | ❌ 不包含 |
| 流向复杂度 | 简单（点对点） | 复杂（多层级） |
| 常见用途 | 地理流向、物流网络 | 能源转换、用户转化 |
