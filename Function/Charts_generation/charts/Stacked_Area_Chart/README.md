# 堆叠面积图

图表分类: 趋势
感知排名: ⭐⭐⭐⭐☆

## 简介

堆叠面积图通过填充区域展示多个数值序列的累积贡献，使用户可以直观观察各部分对整体的贡献以及整体趋势的变化：

- 通过填充区域展示数值大小和累积贡献
- 支持多列堆叠，每一层代表一个数值序列
- 适合展示部分与整体的关系，以及各部分的相对变化

相比折线图，堆叠面积图更适合展示部分与整体的关系；相比堆叠柱状图，堆叠面积图更适合展示连续的时间趋势。


## 数据格式

需要包含以下字段：

- x 列（时间/类别）：X 轴数据，通常为时间序列或类别标签
- y 列（数值）：Y 轴数据，支持单列或多列数值
- series 列（可选）：分组列，用于分组堆叠面积图

或使用**宽格式数据**（自动转换）：

| 时间 | 指标1 | 指标2 | 指标3 |
|------|-------|-------|-------|
| 1月 | 100 | 80 | 60 |
| 2月 | 150 | 120 | 90 |
| 3月 | 120 | 100 | 80 |


## 适用场景

堆叠面积图适用于以下场景：

- 时间序列数据的趋势展示（日、周、月、年数据）
- 多指标堆叠对比（销售额、成本、利润等）
- 部分与整体的关系展示（各地区销售占比、各产品销售贡献）
- 累积变化过程的可视化（库存、用户增长等）


## 典型场景

- 月度销售额趋势：展示不同产品或渠道的销售贡献
- 线上/线下销售对比：堆叠展示两个渠道的销售贡献
- 各部门成本分析：展示不同部门成本的累积变化
- 用户增长趋势：展示新增、活跃、流失用户的变化


## 使用建议

- 数据点建议 ≥ 3 个，以获得良好的视觉效果
- 多列模式下建议 ≤ 5 列，避免图表过于复杂
- 分组模式下建议 ≤ 3 个分组，避免颜色过多导致难以区分
- 时间轴标签较多时可考虑旋转或缩小字体以提高可读性


## 注意事项

- 缺失值会被自动删除，可能导致数据点减少
- 数值列必须可转换为数值类型，否则会被跳过
- 多列模式下自动应用麦肯锡配色方案（10 色循环）
- 宽格式数据会自动转换，第一列作为 x 轴，其余列作为 y 轴数据
- 堆叠面积图中，底部的序列更容易精确读取，中间和顶部的序列由于基线不同会有一定的视觉偏差


## 与相似图表的区别

### 堆叠面积图 vs 折线图

- 堆叠面积图：展示多个数值序列的累积贡献，填充区域使各部分清晰可辨，适合展示部分与整体的关系
- 折线图：强调趋势变化，线条清晰，适合精确对比多个指标

### 堆叠面积图 vs 柱状图

- 堆叠面积图：展示连续的时间序列，视觉上更流畅，强调趋势的连贯性
- 柱状图：展示离散的分类数据，强调各类别间的精确对比

### 堆叠面积图 vs 堆叠柱状图

- 堆叠面积图：适合时间序列，视觉上更流畅，强调累积变化的连贯性
- 堆叠柱状图：适合分类数据，更容易精确读取各部分数值
