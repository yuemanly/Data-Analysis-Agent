# 饼图 Pie Chart

图表分类: 占比 Part-to-Whole  
感知排名: ⭐⭐⭐☆☆

## 简介

饼图是一种占比展示图表，通过扇形的角度编码各部分占整体的比例。它通过圆形分割展示分类数据的占比关系，使用户可以：

- 快速识别最大和最小的部分
- 观察各部分的相对大小关系
- 在演示场景中获得直观的视觉效果

相比柱状图，饼图在类别较少（≤5）时更直观；相比南丁格尔玫瑰图，饼图更简洁但精度更低。

## 数据格式

**必需列**

| 列名 | 类型 | 说明 |
|------|------|------|
| names | 文本/类别 | 扇区名称（如产品、类别） |
| values | 数值(≥0) | 数值大小，用于计算扇形角度 |

**示例数据**

| 产品 | 销售额 |
|------|--------|
| 产品A | 15000 |
| 产品B | 18000 |
| 产品C | 22000 |
| 产品D | 19000 |

## 硬性约束

1. **类别数≤7**（建议≤5）：超过 7 个类别会导致视觉混乱，建议用柱状图
2. 各数值必须 ≥ 0（不支持负数）
3. 人类对角度的感知精度低于长度，不适合精确比较

## 适用场景

✅ **典型场景**
- 市场份额：2-4 个竞争对手的市场占有率
- 投票分布：2-3 个候选人的投票结果
- 简单占比：≤5 个类别的部分整体关系
- 演示场景：视觉冲击力强，适合报告展示

✅ **使用建议**
1. 类别数控制在 2-5 个为佳
2. 数据应按大小降序排列（最大的部分从 12 点钟位置开始）
3. 配合标题和图例清晰表达含义
4. 选择合适的配色方案增强视觉效果

## 注意事项

- 类别过多（>7）会导致视觉混乱，建议用柱状图
- 数值必须 ≥ 0（负数应使用其他图表）
- 人类对角度的感知精度低于长度，不适合精确比较
- 相似大小的扇形难以区分，建议用柱状图
- 轴线/标题/刻度颜色固定为黑色，确保可读性

## 与相似图表的区别

### 饼图 vs 柱状图

| 特性 | 饼图 | 柱状图 |
|------|------|--------|
| 类别数 | 2-5 | 3-30 |
| 精度 | 低 | 高 |
| 视觉冲击 | 强 | 中 |
| 适用 | 简单占比 | 通用对比 |
| 可读性 | 中 | 高 |

### 饼图 vs 南丁格尔玫瑰图

| 特性 | 饼图 | 南丁格尔玫瑰图 |
|------|------|--------|
| 类别数 | 2-5 | 4-12 |
| 编码方式 | 角度 | 面积 |
| 精度 | 低 | 中 |
| 视觉冲击 | 强 | 强 |
| 适用 | 简单占比 | 周期数据 |

### 饼图 vs 甜甜圈图

| 特性 | 饼图 | 甜甜圈图 |
|------|------|--------|
| 中心 | 实心 | 空心 |
| 用途 | 简单占比 | 强调中心信息 |
| 视觉 | 紧凑 | 开放 |
| 中心文字 | 不支持 | 支持 |

## 不适用场景

❌ **不推荐使用**
- 精确数值比较 → 用柱状图或条形图
- 类别过多（>7） → 用柱状图
- 需要精确读数 → 用表格或柱状图
- 相似大小的部分 → 用柱状图
- 负数数据 → 用对比条形图
- 时间趋势 → 用面积图或折线图
- 需要显示中心信息 → 用甜甜圈图
