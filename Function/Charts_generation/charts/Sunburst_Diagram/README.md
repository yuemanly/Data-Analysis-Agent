# 旭日图 Sunburst Diagram

**图表分类**: 占比图 PART-TO-WHOLE  
**感知排名**: ⭐⭐⭐⭐☆

## 简介

旭日图通过圆形分层结构展示多层级的部分与整体关系。与矩形树图类似，旭日图可以展示部分与整体的关系或层级关系，但采用圆形布局。每一圈代表层级中的不同级别，中心圆环显示顶层（根节点），外层圆环展示如何分解为子组。

**核心特点**：
- 用圆形扇区面积表示占比，支持多层级嵌套展示
- 中心圆环为顶层（根节点），外层圆环为子组
- 适合展示 2-3 层层级的分类数据
- 支持交互式钻取和悬停详情
- 可用颜色区分不同层级、组或层级关系

## 数据格式

### 必需列

| 列名  | 类型 | 说明 |
|-------|------|------|
| labels| 文本/类别 | 节点名称，每个节点的唯一标识 |
| values | 数值(>0) | 扇区大小权重，必须为正数 |

### 可选列

| 列名 |类型 | 说明 |
|------|------|------|
| parents | 文本/类别 | 父级标签，用于构建层级关系。父级标签必须在 labels 列中存在 |

### 示例数据

**简单旭日图（无层级）**：

|------|--------|

## 图表选项

当前版本的旭日图暂无可配置选项。图表会自动根据数据特征进行优化：
- 颜色自动按层级或数值大小分配
- 标签过长时自动截断，悬停显示完整内容
- 支持点击钻取（多层级时）

## 适用场景

✅ **典型场景**
- 层级分类展示：地区→城市→门店
- 组织结构展示：公司→部门→团队
- 文件系统展示：磁盘→文件夹→文件
- 预算分配：总预算→部门→项目
- 市场份额：市场→地区→品牌

✅ **使用建议**
1. 数值必须为正数（>0）
2. 行数建议≤200（过多先聚合）
3. 层级深度建议≤3层（过深难以识别）
4. 支持多层级嵌套，通过 parents 列定义层级关系
5. 颜色自动按层级或数值大小分配

## 注意事项

- 数值必须>0：负数或零值会被自动过滤，返回警告
- 层级关系：parents 列中的值必须在 labels 列中存在（形成有效的树结构）
- 性能：数据行数过多（>500）会影响渲染性能，建议先聚合
- 标签显示：标签过长会自动截断，悬停显示完整内容
- 层级深度：过多层级会导致圆环过多，难以识别。建议≤3层

## 与相似图表的区别

### 旭日图 vs 饼图

| 特性 | 旭日图 | 饼图 |
|------|------|------|
| 类别数 | 多（10-100+） | 少（2-7） |
| 层级 | 支持多层级 | 仅单层 |
| 空间利用 | 高效 | 浪费 |
| 精度 | 中（角度难比较） | 低（角度难比较） |
| 用途 | 嵌套分类 | 简单占比 |

### 旭日图 vs 树图

| 特性 | 旭日图 | 树图 |
|------|------|------|
| 形状 | 圆形 | 矩形 |
| 层级 | 支持多层级 | 支持多层级 |
| 空间利用 | 中等 | 高效 |
| 交互 | 点击钻取 | 点击钻取 |
| 用途 | 周期性/圆形数据 | 通用嵌套展示 |

### 旭日图 vs 柱状图

| 特性 | 旭日图 | 柱状图 |
|------|------|--------|
| 类别数 | 多（支持嵌套） | 中等（3-30） |
| 层级 | 支持多层级 | 仅单层 |
| 空间利用 | 高效 | 需要较多空间 |
| 精度 | 中（面积感知） | 高（高度精确） |
| 用途 | 嵌套分类 | 单层对比 |
